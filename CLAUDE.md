## Project Overview

モダンなぷよぷよゲームアプリ — Next.js 16 + React 19 + TypeScript + Tailwind CSS v4 で構築。

## Commands

```bash
npm run dev      # 開発サーバー起動 (http://localhost:3000)
npm run build    # プロダクションビルド
npm run start    # プロダクションサーバー起動
npm run lint     # ESLint 実行
```

## Architecture

**App Router** (`src/app/`) を使用。Pages Router は使わない。

- `src/app/layout.tsx` — ルートレイアウト。Geist フォント、グローバルスタイル適用
- `src/app/globals.css` — Tailwind v4 の `@import "tailwindcss"` と CSS変数でテーマ定義
- `src/app/page.tsx` — ホームページ（Server Component がデフォルト）

### Tailwind CSS v4 の注意点

v4 は設定ファイル不要。`globals.css` の `@theme inline { ... }` ブロックでカスタムトークンを定義する。`tailwind.config.js` は存在しない。

### Next.js 16 / React 19 の注意点

`AGENTS.md` に記載の通り、APIや規約がトレーニングデータと異なる場合がある。`node_modules/next/dist/docs/` 内のガイドを参照すること。

### ゲームロジックの配置方針

- ゲームのコアロジック（盤面、落下、消去判定）は `src/lib/` に純粋関数として実装
- UIコンポーネントは `src/components/` に配置
- ゲーム状態管理には React の `useReducer` を使用し、`src/hooks/` に切り出す

## ぷよぷよ 要件定義

### ゲームの概要

2色〜5色のぷよが上から降ってくる落ち物パズルゲーム。同色のぷよを4つ以上隣接させると消え、連鎖を重ねることでスコアが加算される。

### 機能要件

#### コアゲームルール

- 盤面サイズ: 横6列 × 縦13行（上部2行は非表示の出現バッファ）
- ぷよは2つ1組（ツモ）で上から落下する
- ツモの操作: 左右移動、時計回り／反時計回りの回転、ソフトドロップ（高速落下）、ハードドロップ（即時落下）
- 同色のぷよが4つ以上上下左右に隣接すると消去される
- 消去後に上にあるぷよが落下し、再度消去判定を行う（連鎖）
- 連鎖数に応じてスコアにボーナス倍率がかかる
- 次のツモを1つ先まで表示する（NEXT表示）
- 盤面の出現位置（上部中央）にぷよが積み上がりスペースがなくなるとゲームオーバー

#### スコアリング

- 基本スコア: 消去ぷよ数 × 連鎖ボーナス倍率
- 連鎖ボーナス倍率: 1連鎖=1倍、2連鎖=8倍、3連鎖=16倍、以降+32倍ずつ増加
- ハイスコアをローカルストレージに保存する

#### ゲームモード

- **シングルプレイ** — 制限時間なし、ゲームオーバーまでスコアを稼ぐ
- （将来拡張）対戦モード、タイムアタックモード

### 非機能要件

#### UI/UX

- モダン・先進的なビジュアル: グラスモーフィズムまたはネオモーフィズムを基調としたデザイン
- ぷよはグラデーション付きの円形で表現し、色は赤・青・緑・黄・紫の5色
- 落下・消去・連鎖時にアニメーションを付与（CSS アニメーション or Framer Motion）
- 消去時にパーティクルエフェクトを表示
- レスポンシブ対応: スマートフォン（タッチ操作）とデスクトップ（キーボード操作）の両方をサポート
- ダークモード対応（`prefers-color-scheme` に準拠）

#### キーボード操作

| キー | 動作 |
|------|------|
| ← / → | 左右移動 |
| ↑ | 時計回り回転 |
| Z | 反時計回り回転 |
| ↓ | ソフトドロップ |
| Space | ハードドロップ |

#### タッチ操作（スマートフォン）

- スワイプ左右: 移動
- スワイプ下: ソフトドロップ
- タップ: 回転
- 長押し: ハードドロップ

#### パフォーマンス

- ゲームループは `requestAnimationFrame` ベースで実装し、60fps を維持する
- 落下タイマーは `useReducer` のディスパッチで管理し、React の再レンダリングを最小化する
